## Roles and responsibilities

1. **Back-end engineer** — Designs, implements, and maintains server-side services and APIs. Writes business logic, integrates with databases and caches, ensures service reliability and performance. Participates in code reviews, debugging production issues, and defining service-level agreements (SLAs).

2. **DevOps/SRE (Site Reliability Engineer)** — Manages infrastructure as code, sets up CI/CD pipelines, monitors system health, and handles incident response. Ensures high availability, scalability, and efficient resource utilization. Configures load balancers, container orchestration (e.g., Kubernetes), and observability tools (logging, metrics, tracing).

3. **Security engineer** — Identifies vulnerabilities in the system, designs security protocols (e.g., encryption, authentication), and conducts threat modeling. Implements secure communication protocols (like MTProto), manages secrets and keys, performs security audits and penetration testing, and ensures compliance with data protection standards.

4. **QA engineer** — Develops and executes test plans, writes automated tests (unit, integration, end-to-end), and performs manual testing. Reports and tracks bugs, validates fixes, and ensures that releases meet quality standards before deployment. Works closely with developers to define test coverage and edge cases.

5. **Mobile engineer (iOS/Android)** — Develops and maintains the Telegram mobile applications on iOS and Android platforms. Implements UI/UX designs, integrates with back-end APIs (MTProto), handles offline caching, push notifications, and media playback. Optimizes app performance, battery usage, and network efficiency.

## Common skills across roles

- **Version control (Git)** — All roles use Git for source code management, collaboration, and CI/CD integration.
- **Programming languages** — Proficiency in at least one language relevant to the role (e.g., Go, C++, Python, Swift, Kotlin, Java).
- **API design & integration** — Understanding of REST, RPC, gRPC, or custom protocols (e.g., MTProto) for inter-service and client-server communication.
- **Testing & debugging** — Ability to write tests, read logs, use debuggers and profiling tools to identify and resolve issues.
- **CI/CD & automation** — Familiarity with continuous integration and deployment pipelines, automated testing, and infrastructure automation tools.
- **Monitoring & observability** — Using tools like Prometheus, Grafana, ELK stack, or Datadog to monitor system health, track metrics, and respond to incidents.
- **Security awareness** — Understanding of common vulnerabilities (OWASP), encryption, authentication mechanisms, and secure coding practices.
- **Collaboration & communication** — Working in cross-functional teams, writing documentation, participating in code reviews, and communicating with stakeholders.
